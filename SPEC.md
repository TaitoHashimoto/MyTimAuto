## 1. 概要

### 1.1 目的

Accenture 社内勤怠システム **MyTim** における日々の打刻操作および「**標準労働時間超過報告（残業申請）**」の入力を自動化し、業務効率を改善する個人用ツールセット。

### 1.2 提供する機能

| 機能ID | 機能名                       | 説明                                                                                   |
| ------ | ---------------------------- | -------------------------------------------------------------------------------------- |
| F-01   | 初期セットアップ             | MyTim / Teams への SSO サインインを行い、認証セッションをローカル保存                  |
| F-02   | デスクトップショートカット   | 出勤 / 休憩 / 退勤 用の `.lnk` ファイルを自動生成                                      |
| F-03   | 打刻自動化                   | 出勤・休憩・退勤ボタンを状態判定付きで自動押下                                          |
| F-04   | 残業申請メッセージ検出       | Teams グループチャットから自分の本日分の残業申請メッセージを自動抽出                   |
| F-05   | 残業超過報告フォーム自動送信 | 検出した残業時間・理由を MyTim の「標準労働時間超過報告」フォームに自動入力・送信      |
| F-06   | Windows 通知                 | 各処理の完了・エラーをバルーン通知で表示                                               |
| F-07   | 状態管理・重複送信防止       | 当日送信済み・同一メッセージ再送を防止                                                  |
| F-08   | スケジュール実行             | タスクスケジューラから定期実行可能                                                     |

### 1.3 想定運用シナリオ

```
朝       : デスクトップの「MyTim_出勤.lnk」をダブルクリック → 出勤打刻
昼休み   : 「MyTim_休憩.lnk」をダブルクリック → 休憩打刻（再開時はもう一度 出勤）
日中     : 残業確定後、Teams の所定グループチャットに残業申請メッセージを投稿
夕方     : タスクスケジューラ（平日 15-18時, 15分間隔）が overtime.py を実行
        ↓ 投稿を検出すれば MyTim へ自動報告
        ↓ Windows通知で「残業報告完了 ✓」表示
退勤時   : 「MyTim_退勤.lnk」をダブルクリック → 退勤打刻
```

## 5. 設定ファイル仕様 (`config.py`)

`config.example.py` をコピーして作成。`.gitignore` で除外。

| 変数               | 型     | 必須 | 説明                                                                          |
| ------------------ | ------ | ---- | ----------------------------------------------------------------------------- |
| `MYTIM_URL`        | `str`  | ○    | MyTim 打刻画面のフル URL                                                      |
| `MYTIM_HOST`       | `str`  | ○    | MyTim ホスト名（リダイレクト判定用）                                          |
| `TEAMS_CHAT_ID`    | `str`  | ○    | 対象 Teams グループチャット ID。`19:xxxx@thread.v2` 形式                      |
| `TEAMS_ANCHOR_MSG` | `str`  | ○    | チャット内の任意のメッセージID（チャット起動アンカーとして使用）              |
| `MY_NAME`          | `str`  | ○    | Teams 上の自分の表示名（例: `"Yamada, Taro"`）。送信者フィルタに使用          |
| `MY_EMAIL`         | `str`  | ○    | 自分のメールアドレスのローカルパート                                          |

**読み込み方式**: 全スクリプトで `from config import ...` を行い、見つからない場合は `SystemExit` で起動を停止。

## 6. 機能仕様

### 6.1 `setup.py` — 初期セットアップ

#### 6.1.1 処理フロー

```
1. session/ ディレクトリを作成
2. Playwright で Edge 永続コンテキスト起動（headless=False）
   ├─ channel="msedge"
   └─ Edgeプロファイル切替プロンプト抑止用フラグを付与
3. MyTim URL へ遷移し、ユーザーの SSO サインインを待機（最大5分）
   └─ 完了判定: location.hostname === MYTIM_HOST
4. ネットワークアイドル待機 + 3秒バッファ
5. Teams チャットURL へ別タブで遷移
6. ユーザーへ案内表示（標準出力）
   ├─ パターンA: Edgeプロファイル切替プロンプト → 「サインアウトして別のアカウントで...」をクリック
   ├─ パターンB: 「もう一度サインインする必要があります」バナー → 「サインイン」ボタンをクリック
   └─ パターンC: 既にチャット表示 → そのまま次へ
7. 標準入力で ENTER 待機（asyncio.to_thread 経由）
8. storage_state を storage.json に保存
9. デスクトップに 3 つのショートカットを作成
   ├─ MyTim_出勤.lnk → wscript.exe /nologo 出勤.vbs
   ├─ MyTim_休憩.lnk → wscript.exe /nologo 休憩.vbs
   └─ MyTim_退勤.lnk → wscript.exe /nologo 退勤.vbs
```

#### 6.1.2 Edge 起動フラグ

| フラグ                                                                                                   | 目的                                                  |
| -------------------------------------------------------------------------------------------------------- | ----------------------------------------------------- |
| `--disable-blink-features=AutomationControlled`                                                          | ブラウザ自動化検出を回避                              |
| `--disable-features=msEdgeProfileSwitcher,AccountConsistencyService,IdentityConsistency,EdgeProfileSwitcherFREDialog` | Edge プロファイル切替プロンプトを抑制                  |
| `--no-default-browser-check`                                                                             | 既定ブラウザ確認プロンプト無効                        |
| `--no-first-run`                                                                                         | 初回起動ダイアログ無効                                |
| `--disable-sync`                                                                                         | Edge sync 無効化                                       |

### 6.2 `punch.py` — 打刻スクリプト

#### 6.2.1 コマンドライン引数

```
python punch.py [出勤 | 休憩 | 退勤 | 1 | 2 | 3]
```

| 引数        | 動作   |
| ----------- | ------ |
| `1` / `出勤` | 出勤打刻 |
| `2` / `休憩` | 休憩打刻 |
| `3` / `退勤` | 退勤打刻 |

#### 6.2.2 状態判定ルール

`ALLOWED_STATES` テーブルで現在の打刻状態（未出勤 / 勤務中 / 休憩中）と要求アクションの整合性を判定。

| 現在の状態 → 操作 | 出勤 | 休憩 | 退勤 |
| ---------------- | ---- | ---- | ---- |
| 未出勤            | ✅   | ⛔   | ⛔   |
| 勤務中            | ⛔   | ✅   | ✅   |
| 休憩中            | ✅(=休憩終了) | ⛔   | ⛔   |

`BLOCKED_MESSAGES` テーブルで不許可時の通知メッセージを定義。

#### 6.2.3 状態取得ロジック (`get_state`)

ページ本文 `body.innerText` の以下文字列存在で判定:

```
"休憩中" 含む → 休憩中
"勤務中" 含む → 勤務中
それ以外      → 未出勤
```

#### 6.2.4 ボタン押下ロジック

各アクションに対する **ボタンラベル候補**（多言語対応）:

```python
BUTTONS = {
    "出勤": ["勤務開始", "出勤", "出社", "Clock In"],
    "休憩": ["休憩開始", "休憩", "Break"],
    "退勤": ["勤務終了", "退勤", "退社", "Clock Out"],
}
```

各ラベルに対し以下のセレクタを順に試行:

```
button:has-text('LABEL')
[role='button']:has-text('LABEL')
a:has-text('LABEL')
input[value='LABEL']
span:has-text('LABEL')
```

#### 6.2.5 セッション切れ時の再認証

```
storage.json で起動 → MyTim ホスト到達しない
  → reauth() に遷移
    → Edge 永続プロファイルで再起動（headless=False）
    → ユーザーが SSO 完了するまで最大5分待機
    → 完了後、再度打刻実行 + storage.json 上書き保存
```

### 6.3 `overtime.py` — 残業申請自動送信

#### 6.3.1 コマンドライン引数

| フラグ        | 動作                                                                              |
| ------------- | --------------------------------------------------------------------------------- |
| (なし)        | 通常実行（平日15-18時のみ動作、対象時間外はスキップ）                              |
| `--force`     | 時間帯チェックをスキップ（任意時刻で動作確認）                                     |
| `--dry-run`   | MyTim への送信を行わず、検出メッセージのログ出力のみ                              |

#### 6.3.2 メインフロー (`run`)

```
1. 平日チェック（now.weekday() < 5）。土日 → スキップ（--force で回避）
2. 時間帯チェック（15 <= hour < 18）。範囲外 → スキップ（--force で回避）
3. overtime_state.json から状態をロード
4. 当日既送信チェック（last_report_date == today）。済 → スキップ
5. get_teams_message() で Teams から本日分メッセージを取得
6. 取得失敗 → 終了
7. メッセージのMD5ハッシュを計算
8. 直前送信と同一ハッシュ → スキップ
9. 「残業予定時間：X.Xh」を正規表現で抽出 → hours
10. --dry-run の場合: ログ出力して終了
11. submit_overtime_report(hours, message) でMyTimへ送信
12. 成功時: overtime_state.json を更新
```

#### 6.3.3 Teams メッセージ取得 (`get_teams_message`)

詳細は [§8 メッセージ検出アルゴリズム](#8-メッセージ検出アルゴリズム) 参照。

#### 6.3.4 MyTim 送信 (`submit_overtime_report` / `_submit_on_page`)

```
1. storage.json で headless Edge を起動
2. MyTim URL へ goto
3. ホスト不一致なら reauth_and_submit() へフォールバック
4. ページ内の「標準労働時間超過報告」ボタンをテキスト一致で押下
   ├─ document.querySelectorAll('*') を走査
   ├─ textContent.trim() === "標準労働時間超過報告" の要素を発見
   └─ 見つからない場合は mytim_debug.png にスクリーンショット保存
5. フォーム描画待機（"text=標準労働時間超過報告" を最大10秒）
6. 最後尾の input に hours を入力
7. 最後尾の textarea に reason（整形済みメッセージ）を入力
8. "button:has-text('報告')" の最後尾をクリック
9. 3秒待機 → mytim_after_submit.png に完了スクショ保存
10. 通知「残業報告完了 ✓」を表示
```

## 7. Teams残業申請メッセージ仕様

### 7.1 期待フォーマット

```
日付：YYYY/MM/DD(曜日)
残業予定時間：X.Xh
残業実績時間：X.Xh
残業理由：(任意の理由)
当月深夜勤務累計：X.Xh
当月残業実績累計(休暇込み)：X.Xh
当月休暇実績時間：X.Xh
```

### 7.2 受け入れる表記揺れ

| 項目         | 許容パターン                                                |
| ------------ | ----------------------------------------------------------- |
| 区切り記号   | `：`（全角） / `:`（半角）                                  |
| 日付         | `2026/05/26` / `2026/5/26` （ゼロ埋め有無）                |
| 曜日付記     | `(月)` / `（月）` / 無し                                    |
| 累計フィールド名 | `当月残業実績累計` の後ろに任意文字（`(休暇込み)`等）OK |
| 数値         | 整数 / 小数（例: `1` / `1.5` / `1.0h`）                     |

### 7.3 整形ルール (`clean_overtime_message`)

```
1. \xa0（ノーブレークスペース）、改行、復帰を半角スペースに置換
2. 7つのフィールドに対し各々 re.search で最初の一致を抽出
3. 結果を改行 `\n` で連結、200字で切詰
```

> 200字制限は MyTim の textarea 入力上限（EC1114 エラー回避）。

## 8. メッセージ検出アルゴリズム

### 8.1 課題

[TAISHO]残業連絡 チャットは **複数人が報告するグループチャット**。Teams DOM では送信者ヘッダと本文が別要素のため、単純な「送信者名+本文を含む要素」検索では自分のメッセージを特定しづらい。

### 8.2 検出戦略（優先順）

#### 戦略1: ページ本文テキストでの位置ベース検索（プライマリ）

```
1. document.body.innerText 全体をテキストとして取得
2. 正規表現 /日付[：:]\s*(\d{4})\/(\d{1,2})\/(\d{1,2})/g で全ての日付パターンを列挙
3. 各日付の位置 idx について:
   ├─ body[idx-300 : idx] に MY_NAME が含まれているか確認 → 自分の投稿と判定
   └─ 含まれていなければスキップ
4. 抽出範囲: body[idx : idx + endOffset]
   ├─ endOffset = 1000 を上限
   └─ 「次の日付」が1000文字以内にあれば、そこまでで打ち切り（他人の投稿混入防止）
5. 抽出テキスト内のフィールド数（残業予定時間/残業実績時間/当月休暇実績時間）をカウント
6. 全候補をソート: フィールド数降順 → 位置降順（最新優先）
7. トップ1件を採用
8. message_is_today() で本日付か確認 → 採用 / 却下
```

#### 戦略2: DOM要素ベース検索（フォールバック）

戦略1で見つからなかった場合:

```
1. document.querySelectorAll('*') を走査
2. 要素の innerText に下記すべてが含まれる要素を収集:
   ├─ MY_NAME
   ├─ "日付"
   ├─ "残業予定時間"
   └─ "当月休暇実績時間"
3. 長さ昇順でソートし、minLen + 200 以内の小さい要素群を抽出
4. clean_overtime_message で整形
5. message_is_today() で確認
```

### 8.3 事前処理: Teams 描画安定化

```
1. URL ナビゲート（v2/#/l/message/{chatId}/{anchorMsg}）。3回までリトライ
2. ランチャー経由（dl/launcher）になった場合は URL パラメータから直接 URL を再構築
3. 別タブが開いた場合は ctx.pages 走査で切り替え
4. body.innerText.length > 200 を待機（最大90秒）
5. キャッシュ対策: Teams ホームへ遷移後、対象URLへ再ナビゲート（強制再取得）
6. WebSocket 同期待ち（10秒）
7. 「最新へジャンプ」ボタン検出時はクリック（メッセージペイン内に限定）
8. メッセージスレッド領域を最大8回スクロール（mouse.wheel + End キー）
9. スクリーンショット teams_debug.png 保存
10. 戦略1 → 戦略2 の順に検索
```

### 8.4 セッション切れ検知

検出された最新メッセージが **2日以上前** の日付だった場合、Teams Web の「もう一度サインインする必要があります」バナーが出ている可能性があるとして警告ログ + 通知を発行。バナー自体は Shadow DOM 等の関係で JavaScript からテキストとして取得できないことがあるため、この間接的な検知方式を採用。

## 9. 状態管理

### 9.1 `overtime_state.json`

```json
{
  "last_report_date": "2026-05-26",
  "last_message_hash": "a3f4b5c6d7e8..."
}
```

| キー                | 型     | 説明                                                          |
| ------------------- | ------ | ------------------------------------------------------------- |
| `last_report_date`  | `str`  | 最終送信日（ISO 8601 形式）。当日重複送信防止                 |
| `last_message_hash` | `str`  | 最終送信メッセージの MD5 ハッシュ。同一メッセージ再送防止     |

### 9.2 更新タイミング

- MyTim への送信が成功した時のみ更新
- `--dry-run` 実行時は更新しない
- 送信失敗時（フォーム未検出、セッション切れ等）は更新しない

### 9.3 状態リセット

ファイルを削除すれば次回実行時に空状態で再生成される（手動操作）。

## 10. タイムゾーン

全ての日付・時刻処理は **JST（UTC+9）固定**。

```python
JST = timezone(timedelta(hours=9), name="JST")

def now_jst() -> datetime:
    return datetime.now(JST)

def today_jst() -> date:
    return datetime.now(JST).date()
```

`zoneinfo.ZoneInfo("Asia/Tokyo")` は Windows 標準 Python に `tzdata` が同梱されていないため不採用。

### 適用箇所

| 用途                                  | 関数            |
| ------------------------------------- | --------------- |
| ログタイムスタンプ                    | `now_jst()`    |
| `message_is_today()` の本日判定       | `today_jst()`  |
| 平日 / 時間帯判定 (`run()`)            | `now_jst()`    |
| `overtime_state.json` の `last_report_date` | `today_jst().isoformat()` |
| 警告メッセージの「N日前」計算          | `today_jst()`  |

| Teams 描画タイムアウト               | ログのみ。続行（メッセージ検出失敗となる）                            |

## 12. セキュリティと個人情報の取り扱い

### 12.1 Git 管理から除外する情報

| ファイル / ディレクトリ      | 理由                                                       |
| ---------------------------- | ---------------------------------------------------------- |
| `config.py`                  | 個人/組織固有 ID、メール、表示名                            |
| `session/`                   | Edge プロファイル一式（Cookie、認証トークン、IndexedDB）  |
| `storage.json`               | Playwright storage_state（認証 Cookie）                   |
| `overtime_state.json`        | 送信履歴（個人の業務時刻情報）                              |
| `*.log`                      | Teams メッセージ本文、業務内容、氏名、メールアドレス       |
| `*.png` (スクリーンショット) | Teams チャット内容、MyTim 画面                            |
| `overtime_task.xml`          | ユーザー絶対パスを含む                                     |

### 12.2 認証情報の保管場所

| 認証情報         | 格納先                              | スコープ                                |
| ---------------- | ----------------------------------- | --------------------------------------- |
| Edge セッション   | `session/` (永続プロファイル)        | OS ユーザー固有、暗号化なし（DPAPI 程度） |
| Playwright Cookie | `storage.json` (JSON 平文)          | 読み取り権限を持つ全プロセス             |

> **注**: `storage.json` は平文 JSON のため、ローカルPCの NTFS 権限以外の保護はなし。共有マシンや他者アクセス可能な環境では運用しないこと。

### 12.3 ネットワーク通信

- 全ての通信は HTTPS
- 認証は SSO 経由（パスワードはスクリプトに保存しない）
- 第三者サーバへのデータ送信なし（Microsoft / Accenture 内部システムのみ）

### 12.4 危険な操作のスコープ

スクリプトが行う書き込み操作:
- `MYTIM_URL` 上のフォーム送信のみ（標準労働時間超過報告 / 打刻ボタン）
- Teams への書き込みは **行わない**（読み取りのみ）

## 13. Windows タスクスケジューラ統合

### 13.1 タスク設定

`overtime_task.example.xml` のスキーマに準拠:

| 項目                | 値                                              |

### 13.3 VBS ラッパーの役割

## 14. 既知の制約と注意事項

| # | 制約 | 影響 / 回避策 |
| - | ---- | ----------- |
| 1 | Teams Web セッションが期限切れになると、画面上はキャッシュされた古いメッセージが見え続ける | `setup.py` を再実行 |
| 2 | `headless=True` だと Teams のチャット描画がスキップされる | `overtime.py` は `headless=False` 固定（タスクスケジューラ内でもウィンドウは生成されるが、`InteractiveToken` + `pythonw` でほぼ非表示） |
| 3 | Edge プロファイルが他端末で同アカウントにサインインしていると「プロファイル切替」プロンプトが出る | `--disable-features` フラグで抑制しているが、出た場合は「サインアウトして別のアカウントで...」を選択 |
| 4 | Chrome / Firefox は Accenture Conditional Access で Teams 接続をブロックされる | Edge 固定 |
| 5 | Teams の DOM 構造変更時に検出ロジックが影響を受ける | 位置ベース検索 + フォールバックで一定の耐性。失敗時は `teams_debug.png` で診断可能 |
| 6 | 残業メッセージのフォーマット崩れ（フィールド欠落、数値プレースホルダ `X.Xh` 等） | 該当フィールドが抽出されないだけで他は通る。重要フィールド（残業予定時間）欠落時は送信スキップ |
| 7 | MyTim ボタンのテキスト・配置が変更されると打刻失敗 | 多言語ラベル候補 + 5種類のセレクタで耐性。失敗時は `error_{action}.png` で診断 |
| 8 | `overtime_state.json` を削除すると同一日の二重送信が起こり得る | 通常は削除しない。リセットしたい時のみ手動操作 |
| 9 | 200字制限により残業理由が長文の場合は途中で切れる | MyTim 側仕様。Teams投稿時に簡潔に記載することで回避 |
| 10 | タスクスケジューラの `Repetition` は親トリガーの後でしか動かない（15:00 開始なら最初は 15:15） | 当日 15:00 ちょうどの実行は不要なため許容 |

## 付録 B: 開発履歴の要点

- **JST固定対応**: `datetime.now()` がシステムTZ依存だったため、`timezone(timedelta(hours=9))` で明示
- **複数人チャット対応**: DOM要素ベース検索から、位置ベース（送信者名前提条件付き）に移行
- **キャッシュ問題対応**: 単純ナビゲートだけでは Teams 側がキャッシュ表示を続けるため、Teams ホーム経由の二段階再ナビゲートを実装
- **Edgeプロファイル切替対応**: 起動フラグ + ユーザー操作案内（パターンA/B/C）で複数経路に対応
- **コンプライアンス**: Chrome 試行→Accenture ブロックを受けて Edge 必須に確定
